Automata Theory

A self-contained reference covering regular languages, finite automata, and the subset construction algorithm. Start here if you are new to the subject, or use it as a refresher alongside the designer.

1. Regular Languages

A language is simply a set of strings over some alphabet Σ. We say a language is regular if it can be recognised by a finite automaton — equivalently, if it can be described by a regular expression, or generated by a right-linear grammar. All three descriptions capture exactly the same class of languages (Kleene's theorem).

Definition — Regular Language

A language L ⊆ Σ* is regular if and only if there exists a deterministic finite automaton (DFA) M such that L(M) = L.

Closure properties

The class of regular languages is closed under union, concatenation, Kleene star, complement, intersection, and reversal. This means that combining regular languages with these operations always produces another regular language — a fact that makes them very tractable.

Limits

Not every language is regular. The classic non-example is { a^n b^n | n ≥ 0 } — strings of equal numbers of a's and b's. The pumping lemma provides the standard tool for proving a language falls outside this class: it states that every sufficiently long string in a regular language has a "pumpable" middle section.

2. Regular Expressions

A regular expression (regex) is a compact notation for describing a regular language using three primitive operations and parentheses for grouping.

The three primitive operations

Concatenation — ab matches the string "ab".
Union (alternation) — a|b matches "a" or "b".
Kleene star — a* matches the empty string, "a", "aa", "aaa", …

Base cases

  • The empty language ∅ is a regular expression.
  • The empty string ε is a regular expression.
  • Each symbol a ∈ Σ is a regular expression.

Inductive cases

If R and S are regular expressions, then so are (R|S), (RS), and (R*). Precedence (high to low): star, then concatenation, then union — so ab*|c parses as (a(b*))|c.

Examples

  • (0|1)* — all binary strings, including the empty string.
  • (0|1)*1(0|1)* — binary strings containing at least one 1.
  • a*ba*ba* — strings over {a, b} with exactly two b's.
  • (ab)* — even-length strings alternating a and b.

3. Nondeterministic Finite Automata (NFA)

An NFA is a theoretical machine with a finite set of states that reads an input string one symbol at a time and transitions between states. Unlike a DFA, it may have multiple possible transitions for the same symbol from a given state, no transition at all for some symbol, or spontaneous ε-transitions that consume no input.

Formal definition

An NFA is a 5-tuple M = (Q, Σ, δ, q₀, F) where:

Q    — finite set of states
Σ    — finite input alphabet
δ : Q × (Σ ∪ {ε}) → 2^Q    — transition function (returns a set of states)
q₀ ∈ Q    — start state
F ⊆ Q    — set of accepting (final) states

Acceptance

An NFA accepts a string w if there exists at least one sequence of transitions — including ε-moves — that leads from q₀ to some state in F after reading all of w. It rejects w if no such sequence exists.

The epsilon-closure

The ε-closure of a set of states S, written ε-closure(S), is the set of all states reachable from any state in S via zero or more ε-transitions. It is computed with a simple graph reachability search (BFS or DFS) following only ε-labelled edges.

Why NFAs?

NFAs are often much easier to construct from a regular expression — the Thompson construction converts any regex to an NFA in linear time. They are a convenient intermediate representation, even though execution requires simulation of all possible paths in parallel.

4. Deterministic Finite Automata (DFA)

A DFA is the simplest model of computation that recognises exactly the regular languages. At every step, given the current state and the current input symbol, there is exactly one next state — hence "deterministic".

Formal definition

A DFA is a 5-tuple M = (Q, Σ, δ, q₀, F) where:

Q    — finite set of states
Σ    — finite input alphabet
δ : Q × Σ → Q    — transition function (returns exactly one state)
q₀ ∈ Q    — start state
F ⊆ Q    — set of accepting states

Acceptance

A DFA accepts string w = a₁a₂...aₙ if the unique path q₀ → q₁ → ... → qₙ defined by repeated application of δ ends in a state qₙ ∈ F.

Completeness

A DFA is called complete (or total) when δ is defined for every (state, symbol) pair. When the subset construction creates a DFA from an NFA, any state with no NFA counterpart is mapped to a single dead state (trap state) from which no accepting state is reachable.

Minimisation

Every regular language has a unique minimal DFA (up to renaming of states). The table-filling algorithm (Myhill–Nerode) finds equivalent states and merges them, producing the smallest possible DFA for a given language.

5. Converting an NFA to a DFA

The subset construction (also called the powerset construction) is the standard algorithm for converting any NFA into an equivalent DFA. The key insight is that the DFA simulates all possible paths of the NFA in parallel: each DFA state represents a set of NFA states the machine could currently be in.

Intuition

When an NFA reads a symbol from state q, it may move to several states at once. Instead of choosing one, we track all of them. The resulting "set of states" becomes a single DFA state. After reading the full input, the DFA accepts if any NFA state in the current set is a final state.

The algorithm

Subset Construction — Step by Step
  1. Compute the ε-closure of the NFA start state. This becomes the DFA start state, labelled with that set, e.g. {q0, q1}.
  2. Add this set to the worklist of unprocessed DFA states.
  3. Take a DFA state S from the worklist. For each alphabet symbol a ∈ Σ:
    • Compute move(S, a) — the set of NFA states reachable from any state in S via an a-transition.
    • Compute ε-closure(move(S, a)).
    • If the result is a new (unseen) set, add it to the DFA state set and the worklist.
    • Record the DFA transition S → ε-closure(move(S, a)) on symbol a.
  4. Repeat step 3 until the worklist is empty.
  5. Mark a DFA state as accepting if its NFA-state set contains at least one NFA final state.
  6. Any symbol from a DFA state with no outgoing transition leads to a dead state — a non-accepting state that loops on all symbols.

Complexity

In the worst case, an NFA with n states can produce a DFA with up to 2^n states (one for each subset of Q). In practice, most subsets are unreachable and the DFA is far smaller. The exponential blowup is unavoidable in the worst case — there exist families of NFAs for which it is achieved.

Equivalence theorem

Theorem (Rabin & Scott, 1959)

For every NFA N there exists a DFA D such that L(N) = L(D). Therefore NFAs and DFAs recognise exactly the same class of languages — the regular languages.
